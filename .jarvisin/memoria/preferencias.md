# ⚙️ Preferencias Operativas

```yaml
# ─── ESTILO DE COMUNICACIÓN ──────────────────────────────────
respuesta_esperada: Directa y orientada a acción inmediata
formato_preferido: Listas cortas, tablas, bloques de código cuando aplica
nivel_detalle: Medio — no demasiado corto ni demasiado largo
idioma_respuesta: Español (siempre)
evitar: Preguntas innecesarias, repetir lo que ya se sabe, relleno sin valor

# ─── HERRAMIENTAS DEL ECOSISTEMA ─────────────────────────────
vault: Obsidian (metodología PARA)
control_versiones: Git + GitHub (flujo: pull → work → push)
musica_ia: Suno AI v5.5
distribucion_musical: Symphonic
bd_preferida: PostgreSQL / Supabase
frontend_preferido: React / Next.js
editor_codigo: VSCode
modelo_ia: Claude (como Jarvisin)
modelos_alternativos: OpenRouter (pool de modelos gratuitos como fallback)

# ─── FLUJO DE TRABAJO ─────────────────────────────────────────
metodologia: PARA (Projects, Areas, Resources, Archive)
ciclo_git: Obligatorio al inicio y final de cada sesión
bitacora: Siempre actualizar después de cambios significativos
formato_bitacora: "[YYYY-MM-DD] - @usuario: Cambio. Impacto: Resultado."
ciclo_lanzamiento_musicall: 1 single cada ~7 días via Symphonic

# ─── QUÉ LE GUSTA ─────────────────────────────────────────────
le_gusta:
  - Sistemas bien documentados y escalables
  - IA que actúa sin pedir permiso para cada micro-decisión
  - Entregas que ya incluyen el "siguiente paso" sugerido
  - Brevedad con sustancia (no verbosidad vacía)

# ─── QUÉ NO LE GUSTA ─────────────────────────────────────────
no_le_gusta:
  - Respuestas que solo reformulan la pregunta sin avanzar
  - Preguntas para cosas que ya están documentadas en el Vault
  - Código comentado en exceso o con docstrings innecesarios
  - Proponer hacer algo sin hacerlo directamente
```
